Create the shared grep script that both CI and the pre-commit hook (Plan 05) will use. Verbatim from RESEARCH.md §"Code Examples → Secret-Leak Grep Script", with one Windows-friendly tweak (the script runs in Git Bash on Windows and in Ubuntu on CI — identical bash):

```bash
#!/usr/bin/env bash
# scripts/secret-leak-grep.sh
# Used by: .git/hooks/pre-commit (installed by Plan 05) AND .github/workflows/pr-checks.yml
# Enforces INFRA-04 + SEC-07: service-role / SERVICE patterns must NEVER appear in src/.

set -euo pipefail

PATTERNS=(
  'VITE_[A-Z_]*SERVICE'
  'service_role'
  'SUPABASE_SERVICE'
  'SUPABASE_SERVICE_ROLE_KEY'
)

# Build PCRE alternation
COMBINED=$(IFS='|'; echo "${PATTERNS[*]}")

# Scope: src/ only. Exclude test files (mock values in tests are allowed but should be reviewed manually).
# rg exits 0 if matches found (BAD), 1 if no matches (GOOD), 2 if error.
if rg --pcre2 --hidden \
     --glob '!*.test.*' \
     --glob '!*.spec.*' \
     --type-add 'srccode:*.{ts,tsx,js,jsx,html,css}' \
     -t srccode "$COMBINED" src/; then
  echo ""
  echo "ERROR: Possible secret leak detected in src/."
  echo "Forbidden patterns: ${PATTERNS[*]}"
  echo "Service-role keys must NEVER appear in src/."
  echo "Move them to:"
  echo "  - Supabase Edge Function env vars (for runtime)"
  echo "  - GitHub Actions secrets (for CI)"
  exit 1
fi

echo "Secret-leak grep: clean."
exit 0
```

Make executable:
```powershell
git update-index --chmod=+x scripts/secret-leak-grep.sh
```
(On Windows, this sets the executable bit in git's index so the Ubuntu CI runner sees the file as executable.)

**Note on dependencies:** The script uses `rg` (ripgrep). On Ubuntu CI runners ripgrep is pre-installed. For local Windows dev, founder must `scoop install ripgrep` — Plan 05 documents this in the pre-commit setup script.

**File 1: `.github/workflows/pr-checks.yml`** — the 5 PR checks (INFRA-03):

```yaml
name: PR Checks
on:
  pull_request:
    branches: [main]

permissions:
  contents: read

jobs:
  lint-typecheck-build:
    name: Lint + Typecheck + Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: 'package.json'
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npx tsc -b
      - run: npm run build
        env:
          # Dummy DSN — build must not fail when Sentry env absent
          VITE_SENTRY_DSN: ''

  secret-leak-grep:
    name: Secret Leak Grep (INFRA-04 / SEC-07)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: bash scripts/secret-leak-grep.sh

  supabase-dry-run:
    name: Supabase Migration Dry-Run (INFRA-03)
    runs-on: ubuntu-latest
    env:
      SUPABASE_ACCESS_TOKEN: ${{ secrets.SUPABASE_ACCESS_TOKEN }}
      SUPABASE_DB_PASSWORD: ${{ secrets.STAGING_DB_PASSWORD }}
    steps:
      - uses: actions/checkout@v4
      - uses: supabase/setup-cli@v1
        with:
          version: 2.84.5
      - run: supabase link --project-ref ${{ secrets.STAGING_PROJECT_ID }}
      - run: supabase db push --linked --dry-run

  npm-audit:
    name: npm audit (INFRA-08)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: 'package.json'
          cache: 'npm'
      - run: npm ci
      - run: npm audit --audit-level=high
```

**File 2: `.github/workflows/deploy-staging.yml`** — defense-in-depth dry-run on PR open (Supabase docs recommend this even though pr-checks.yml also runs it; the duplicate gives clearer error attribution):

```yaml
name: Deploy migrations to staging (dry-run on PR, apply on develop)
on:
  pull_request:
    branches: [main]
    paths:
      - 'supabase/migrations/**'
  push:
    branches: [develop]
    paths:
      - 'supabase/migrations/**'

jobs:
  deploy-staging:
    runs-on: ubuntu-latest
    env:
      SUPABASE_ACCESS_TOKEN: ${{ secrets.SUPABASE_ACCESS_TOKEN }}
      SUPABASE_DB_PASSWORD: ${{ secrets.STAGING_DB_PASSWORD }}
    steps:
      - uses: actions/checkout@v4
      - uses: supabase/setup-cli@v1
        with:
          version: 2.84.5
      - run: supabase link --project-ref ${{ secrets.STAGING_PROJECT_ID }}
      - name: Dry run on PR
        if: github.event_name == 'pull_request'
        run: supabase db push --linked --dry-run
      - name: Apply on develop push
        if: github.event_name == 'push' && github.ref == 'refs/heads/develop'
        run: supabase db push --linked
```
